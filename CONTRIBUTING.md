# Contributing

Thanks for your interest in contributing!

## How to Contribute

### Report Issues
- Found a bug or unclear documentation? Open an issue.
- Describe what you expected vs. what happened.

### Suggest Skills
- Have an idea for a useful skill? Open an issue with:
  - Skill name and purpose
  - Trigger keywords
  - Expected workflow

### Submit Pull Requests
1. Fork the repo
2. Create a feature branch: `git checkout -b feature/my-skill`
3. Add your changes
4. Test with Claude Code
5. Submit a PR with a clear description

## Guidelines

- Keep skills focused — one skill, one purpose
- Document your skills with clear triggers and workflows
- Use the existing skill format (see `docs/skill-development.md`)
- English for code and docs, bilingual welcome for user-facing content

## Code of Conduct

Be respectful, constructive, and helpful. We're all learning together.
